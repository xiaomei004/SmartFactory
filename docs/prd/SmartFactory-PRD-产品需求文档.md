## 1. 产品概述

### 1.1 产品定位

SmartFactory（东软智能制造云平台）是一个基于**工业4.0**理念的**云工厂智能制造管理平台**。系统模拟真实工厂从接单、排产、调度、生产到报工的全链路管理流程，为工厂管理者提供一站式生产数字化管理能力。

### 1.2 核心价值

| 价值维度 | 描述 |
|----------|------|
| **集中数据管理** | 工厂、产品、设备、订单、计划、工单数据统一管理，信息实时共享 |
| **全流程在线化** | 从订单接收到生产完成的完整工作流在线处理，无需线下流转 |
| **实时状态可视** | 首页仪表盘实时展示设备运行状态、订单进度、生产效率指标 |
| **产能精准匹配** | 接单时自动校验可用产能，防止超接导致交付延期 |
| **可扩展架构** | 预留对接设备操控、产能交易、电商接单、物流发货等系统的扩展能力 |

### 1.3 核心术语

| 术语 | 英文 | 说明 |
|------|------|------|
| **工厂** | Factory | 入驻平台的制造工厂，所有业务数据的隔离单元 |
| **产品** | Product | 工厂可生产的产品，需提前注册 |
| **设备** | Equipment | 工厂的生产设备，可关联可生产产品及产能 |
| **订单** | Order | 来自线上/线下的生产订单 |
| **生产计划** | Production Plan | 基于已接订单生成的排产计划 |
| **生产调度/工单** | Schedule / Work Order | 将计划拆分至具体设备的工单 |
| **报工** | Daily Work Report | 设备生产情况的汇报记录 |
| **产能** | Capacity / Yield | 设备在单位时间内可生产某产品的数量 |

### 1.4 产品边界

**包含**：工厂注册、产品管理、设备管理、订单管理、生产计划管理、生产调度管理、生产跟踪报工、首页仪表盘

**不包含（V1.0）**：设备远程操控、产能交易市场、电商对接、物流发货、高级数据分析/BI、移动端App

## 2. 用户角色与权限

### 2.1 角色定义

| 角色 | 职责 | 核心权限 |
|------|------|----------|
| **系统管理员** | 平台运维，工厂注册审批 | 用户管理、工厂管理、权限配置 |
| **订单管理员** | 订单的创建、接单/拒单、转计划 | 订单管理（CRUD + 状态流转） |
| **计划管理员** | 将订单转化为生产计划并排期 | 生产计划管理（CRUD + 启动） |
| **调度管理员** | 将计划拆分为工单并分配设备 | 生产调度管理（CRUD + 启动） |
| **生产员工** | 执行生产任务并报工 | 生产跟踪（报工操作、完成报工） |

### 2.2 权限模型

```
用户 (t_user) ──┬── 工厂 (t_factory)     ← 数据隔离边界
                └── 角色 (t_user_role)    ← 功能权限聚合
                      └── 权限 (t_permit) ← 细粒度操作权限
```

- **数据隔离**：所有业务操作以 `factory_id` 为基础隔离条件
- **默认规则**：注册工厂的用户自动成为该工厂管理员

### 3.4 模块三：首页仪表盘

| 属性 | 说明 |
|------|------|
| **优先级** | 高 |
| **裁剪** | 可选 |
| **功能描述** | 登录后首屏，动态展示工厂运行概况 |

**展示板块**：

#### a) 设备运行情况
| 指标 | 计算规则 |
|------|----------|
| 加工中设备数 | 当前已启动工单中匹配的设备数 |
| 待机中设备数 | 未启动工单匹配的设备数 |
| 停机中设备数 | 状态为"停用"的设备数 |
| **开机率** | (加工中 + 待机) ÷ 全部设备 × 100% |
| **故障率** | 故障设备 ÷ 全部设备 × 100% |
| **运行率** | 加工中 ÷ 全部设备 × 100% |
| **综合效率** | 加工中 ÷ 已启用设备 × 100% |

#### b) 设备信息列表
- 展示所有设备的当前运行状态和信息概览
- 数据源：设备管理模块

#### c) 当前订单数据统计
- 展示订单跟踪状况
- 数据源：订单管理模块

#### d) 年度订单情况汇总
- 各月份已接单、已排产、已完成订单数量统计
- 数据源：订单管理模块

### 3.6 模块五：设备管理

| 属性 | 说明 |
|------|------|
| **优先级** | 高 |
| **裁剪** | 不可裁剪（核心基础数据） |
| **功能描述** | 管理工厂设备及设备-产品产能关系 |

**功能操作**：

```
┌──────────────────────────────┐
│  设备列表页                   │  ← 支持按产品名称、设备名称搜索
├──────────────────────────────┤
│  [新增设备]                   │  → 填写设备信息 + 关联可生产产品及产能
│  [编辑设备]                   │  → 修改设备信息，序列号不可重复
│  [删除设备]                   │  → 已关联已启动工单时不可删除
└──────────────────────────────┘
```

**业务规则**：
- 设备序列号必填，同一工厂内不可重复
- 设备状态：`10-启用` / `20-停用` / `30-故障`
- 同一设备对同一产品只能有一条产能记录
- 只有关联了可生产产品并设置了产能的设备才能参与调度
- 已关联已启动工单的设备不可删除

**字段定义**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 设备序列号 | String | 是 | 唯一 |
| 设备名称 | String | 否 | |
| 设备状态 | Enum | 是 | 启用/停用/故障 |
| 设备图片 | File/URL | 否 | |
| 可生产产品 | Select | 否 | 关联产品管理，支持多选 |
| 产能 | Integer | 否 | 与产品对应，单位：天/月/年/小时 |

### 3.7 模块六：订单管理

| 属性 | 说明 |
|------|------|
| **优先级** | 高 |
| **裁剪** | 不可裁剪（核心业务流程） |
| **功能描述** | 订单的完整生命周期管理 |

**订单状态流转**：

```
  新建(线下订单)
       │
       ▼
 ┌─────────┐    接单    ┌─────────┐   转计划   ┌─────────┐   完成   ┌─────────┐
 │ 未接单   │ ────────→ │ 已接单   │ ────────→ │ 生产中   │ ──────→ │ 已完成   │
 └─────────┘           └─────────┘           └─────────┘         └─────────┘
       │                    │
       │ 拒单               │ (仅有未关联计划时可拒单)
       ▼                    ▼
 ┌─────────┐           (不可逆)
 │ 已拒绝   │
 └─────────┘
 (终态，不可再操作)
```

**操作说明**：

| 操作 | 前置条件 | 效果 |
|------|----------|------|
| **新建** | - | 创建线下订单，状态=未接单 |
| **接单** | 可用产能 ≥ 订单数量 | 状态=已接单；若产能不足则提示 |
| **拒单** | 填写拒绝备注 | 状态=已拒绝（终态） |
| **转计划** | 状态=已接单 | 填写计划起止日期，生成未启动的生产计划，订单状态→生产中 |
| **完成** | 完成数量 ≥ 订单数量 | 状态=已完成；若完成数量不足，需填备注方可完成。关联的计划和工单同步完成 |

**业务规则**：
- 只有注册了产品的工厂才能处理订单
- 接单时自动计算可用产能，产能不足给出明确提示
- 订单完成后，关联的所有生产计划和工单自动置为已完成
- 订单编号后台自动生成（格式：O + 时间戳，如 `LG001`）

**字段定义**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 订单编号 | String | 否(自动) | O开头+时间戳 |
| 订单来源 | Enum | 否 | 默认为线下订单 |
| 产品名称 | Select | 是 | 关联产品管理 |
| 产品数量 | Integer | 是 | |
| 订单截止日期 | Date | 是 | |
| 订单状态 | Enum | 否(自动) | 未接单→已接单→生产中→已完成 / 已拒绝 |
| 已完成数量 | Integer | 否 | 系统自动汇总 |
| 可用产能 | Integer | 否 | 系统自动计算 |

### 3.8 模块七：生产计划管理

| 属性 | 说明 |
|------|------|
| **优先级** | 高 |
| **裁剪** | 可裁剪（简化为直接调度） |
| **功能描述** | 针对已接单订单创建和管理排产计划 |

**计划状态流转**：

```
 ┌──────────┐   启动   ┌──────────┐   (所有工单完成)   ┌──────────┐
 │  未启动   │ ──────→ │  已启动   │ ────────────────→ │  已完成   │
 └──────────┘         └──────────┘                    └──────────┘
      │                     │
   可编辑/删除          不可编辑/删除
                      自动生成未开始工单
```

**操作说明**：

| 操作 | 前置条件 | 效果 |
|------|----------|------|
| **新建** | 订单状态=生产中 | 填写计划起止日期，创建未启动计划 |
| **启动** | 计划状态=未启动 | 状态→已启动，自动生成一条设备为空的未开始工单 |
| **删除** | 计划状态=未启动 | 删除计划；每条订单至少保留一条生产计划 |

**业务规则**：
- 计划来源于已接单的订单
- 启动计划时自动创建一条未分配设备的工单
- 已启动计划不可编辑、不可删除
- 每条"生产中"订单必须保持至少一条生产计划

**字段定义**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 计划编号 | String | 否(自动) | 后台自动生成 |
| 订单编号 | Select | 是 | 关联已接单订单 |
| 产品 | String | 否(自动) | 选择订单后自动带出 |
| 计划数量 | Integer | 是 | 选择订单后自动带出 |
| 交货日期 | Date | 否 | 选择订单后自动带出 |
| 计划开始时间 | Date | 是 | |
| 计划结束时间 | Date | 是 | |
| 计划状态 | Enum | 否(自动) | 未启动/已启动/已完成 |

### 3.9 模块八：生产调度管理

| 属性 | 说明 |
|------|------|
| **优先级** | 高 |
| **裁剪** | 可裁剪 |
| **功能描述** | 将已启动的生产计划拆分为具体工单并分配设备 |

**工单状态流转**：

```
 ┌──────────┐   启动   ┌──────────┐   报工完成   ┌──────────┐
 │  未开始   │ ──────→ │  生产中   │ ──────────→ │  已完成   │
 └──────────┘         └──────────┘              └──────────┘
      │                     │
   可编辑/删除          不可编辑/删除
```

**操作说明**：

| 操作 | 前置条件 | 效果 |
|------|----------|------|
| **新建工单** | 计划状态=已启动 | 选择设备（仅限可生产该产品的设备），创建未开始工单 |
| **启动工单** | 工单状态=未开始 | 状态→生产中，开始生产 |
| **删除工单** | 工单状态=未开始 | 删除；每条计划至少保留一条工单 |

**业务规则**：
- 工单建立在已启动的生产计划基础上
- 只有可生产该产品的设备才能被选择
- 已启动工单不可编辑、不可删除
- 每条已启动计划必须保留至少一条工单

**字段定义**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 工单编号 | String | 否(自动) | 后台自动生成 |
| 计划编号 | Select | 是 | 关联已启动计划 |
| 产品 | String | 否(自动) | 选择计划后自动带出 |
| 产品数量 | Integer | 否(自动) | 自动带出 |
| 计划数量 | Integer | 否 | 来源于生产计划 |
| 设备序列号 | Select | 是 | 仅显示可生产该产品的设备 |
| 开始日期 | Date | 是 | |
| 结束日期 | Date | 是 | |
| 工单状态 | Enum | 否(自动) | 未开始/生产中/已完成 |

### 3.10 模块九：生产跟踪（报工）

| 属性 | 说明 |
|------|------|
| **优先级** | 高 |
| **裁剪** | 不可裁剪（核心闭环） |
| **功能描述** | 生产员工对设备生产情况进行报工，记录加工数据，推进生产进度 |

**操作说明**：

| 操作 | 前置条件 | 效果 |
|------|----------|------|
| **报工** | 工单状态=生产中 | 录入加工数量、合格数量、加工起止时间 |
| **完成报工** | 报工时勾选"结束报工" | 工单→已完成，更新订单已完成数量 |

**业务规则**：
- 完成报工时自动累计更新订单的已完成数量
- 订单完成数量达到订单数量时，可触发订单完成操作
- 若未达订单数量但需完成，须填写备注

**字段定义**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 工单编号 | Select | 是 | 关联生产中工单 |
| 产品名称 | String | 否(自动) | 自动带出 |
| 设备序列号 | String | 否(自动) | 自动带出 |
| 工单状态 | Enum | 否(自动) | |

## 9. 业务流程图

### 9.1 整体业务流程

```
[注册工厂] → [录入产品] → [录入设备→关联产品→设置产能]
                                │
                                ▼
[新建订单] → [接单(产能校验)] → [转生产计划] → [计划启动(生成空工单)]
                                                    │
                                                    ▼
                              [工单分配设备] → [工单启动] → [生产报工]
                                                                │
                                                                ▼
                              [完成报工(更新订单进度)] → [订单完成(关联计划/工单全部完成)]
```

### 9.2 订单全生命周期

```
订单来源 (线上/线下)
    │
    ├── 新建 → 未接单
    │           ├── 接单(产能OK) → 已接单 → 转计划 → 生产中 → 完成 → 已完成
    │           └── 拒单(填备注) → 已拒绝
    │
    └── 订单完成条件：已完成数量 ≥ 订单数量
```

### 9.3 生产执行链路

```
订单(已接单) → 生产计划(未启动) → [启动] → 生产计划(已启动) → 自动生成工单(未开始,无设备)
                                                              │
                                    ┌─────────────────────────┘
                                    ▼
                              分配设备 → 工单(未开始) → [启动] → 工单(生产中)
                                                                      │
                                                                      ▼
                                                                  报工(录入数据)
                                                                      │
                                                          ┌───────────┘
                                                          ▼
                                                     [完成报工] → 工单(已完成)
                                                          │
                                                    更新订单已完成数量
```
